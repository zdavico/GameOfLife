# Game of Life Budget Dashboard

A React-based budget entry dashboard for the Game of Life event, a first-year financial literacy milestone in Champlain College's InSight program. Money managers use this tool to enter student budget data during the live event, visualize 50/30/20 allocation in real time, and generate coaching summaries.

Built with React + Vite + Recharts. Deployed to GitHub Pages.


## Features

- Student lookup by name, ID, or email with instant search
- Budget entry across 11 categories matching the Game of Life budget sheet
- 50/30/20 analysis with real-time needs/wants/goals calculation
- Four visualizations: animated donut comparison, Sankey income flow, waterfall chart, and radar chart
- Zero-based budget thermometer tracking allocation toward income
- "What If" sliders for interactive coaching conversations
- Coaching prompts that surface when categories exceed suggested baselines
- Print/share summary for students to screenshot or print
- Aggregate dashboard with post-event reporting across all processed students
- Dark and Champlain light themes with adjustable font size
- Offline-capable: student data is baked into the build, no network needed on event day

## Quick Start

### Prerequisites

- **Node.js** 18+ (`node --version` to check)
- **npm** (bundled with Node)
- **Git**

### 1. Clone the project

```bash
git clone https://github.com/YOUR_USERNAME/GameOfLife.git
cd GameOfLife
```

Or if starting from a zip, unzip it and open the folder in PyCharm.

### 2. Install dependencies

Open PyCharm's terminal (Alt+F12, or View > Tool Windows > Terminal):

```bash
npm install
```

### 3. Add your student data

Export the **Master** tab from the Master Money Manager Google Sheet as a CSV:

1. Open the sheet in Google Sheets
2. Go to the **Master** tab
3. File > Download > Comma Separated Values (.csv)
4. Rename the file to `master.csv`
5. Move it into the `data/` folder

### 4. Build the student data

```bash
npm run build-data
```

Expected output:

```
Reading: data/master.csv
  Parsed 399 rows
  Found 325 unique students (skipped 74 non-student rows)
  Wrote 325 students to src/data/students.json
  Done! Run 'npm run dev' to see the dashboard.
```

If you skip this step, the app loads with 12 sample students.

### 5. Run the dev server

```bash
npm run dev
```

Opens at http://localhost:5173 with hot reload.

### 6. Test it

1. Search for a student in the search bar
2. Enter dollar amounts in the budget categories
3. Watch the visualizations update live
4. Toggle "What If" sliders after entering values
5. Click "Save & Next" to log the student to the session
6. Switch to the "Aggregate" tab to see reporting


## Deploying to GitHub Pages

### One-time setup

1. Create a repo at [github.com/new](https://github.com/new) named `GameOfLife` (don't initialize with a README)

2. Confirm the `base` path in `vite.config.js` matches your repo name:
   ```js
   base: '/GameOfLife/',
   ```

3. Push your code:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/GameOfLife.git
   git push -u origin main
   ```

4. Deploy:
   ```bash
   npm run deploy
   ```

5. In your repo on GitHub, go to Settings > Pages. Under "Source," select the `gh-pages` branch and `/ (root)`. Click Save.

6. Your site will be live at:
   ```
   https://YOUR_USERNAME.github.io/GameOfLife/
   ```

### Updating the deployment

```bash
npm run build-data    # only if CSV changed
npm run deploy        # build + push to gh-pages
```

If you also changed code and want to keep your `main` branch current:

```bash
git add .
git commit -m "Description of changes"
git push
npm run deploy
```


## Student Data Privacy

`data/master.csv` and `src/data/students.json` contain student PII (names, emails, IDs, financial data).

- `data/master.csv` is **gitignored** by default and never committed
- `src/data/students.json` is **not** gitignored so it can deploy to GitHub Pages
- If your repo is **public**, uncomment the `src/data/students.json` line in `.gitignore` and switch to a private repo instead
- For a **private repo**, the default setup is fine

For safe testing: anonymize sID, first name, and last name in `master.csv`, and use localhost or a private repo.


## Updating Student Data

When the roster changes or income data is updated:

1. Re-export the Master sheet as CSV
2. Replace `data/master.csv`
3. Run `npm run build-data`
4. Run `npm run deploy` to update the live site

The build script handles deduplication, tax estimation, and sorting automatically.

## Customization

### Budget categories

Edit the `CATEGORIES` array in `src/components/GameOfLifeDashboard.jsx`. Each entry has a `key`, `label`, `hint`, `alloc` (needs/wants/goals split, must sum to 1.0), and `suggested` (baseline percentage for coaching prompts).

### Coaching prompts

Edit the `COACHING` object in the same file. Each key matches a category key.

### Themes

Edit the `THEMES` object. The `dark` and `light` themes define every color used in the dashboard.

### Column mapping

If your CSV has different column headers, check the `findCol` function in `scripts/build-data.js` and add your column names to the candidates list.
